1. Brainstormed Features and Elements

Main landing page displaying all team members
Clickable avatars/images for each team member
Individual profile pages for each member
Basic navigation (home ↔ profile pages)
Consistent layout and styling using CSS
Personal avatar image for each member
Separate CSS file for each profile page
Organized folder structure (docs, css, images, etc.)
Responsive layout (optional if beginner level)
Short bio or description on profile pages

2. User Stories for Each Feature

As a user, I want to see all team members on one page so that I know who is in the group.
As a user, I want to see images of each member so that I can recognize them visually.
As a user, I want to click on a team member’s image so that I can view their profile.
As a user, I want each profile page to show information about the person so that I can learn more about them.
As a user, I want consistent styling so that the website looks clean and professional.
As a user, I want easy navigation so that I can move between pages without confusion.
As a user, I want the site to load properly so that I can access all content without errors.

3. MVP (Minimum Viable Product) Clearly Defined

The MVP for this project includes:

A working index.html landing page
Display of all team members with images
Clickable images linking to individual profile pages
Individual profile pages for each team member
Basic CSS styling applied using css/style.css
Proper folder structure (HTML, CSS, images, docs)
Each profile includes:
Avatar image
Name
Basic information

4. Categorization of Features

Need to Have (MVP)

Landing page with team members
Clickable avatars
Individual profile pages
Basic CSS styling
Organized project folder
Avatar images for each member

Nice to Have

Better styling (colors, fonts, hover effects)
Responsive design for mobile
Additional info (skills, links, fun facts)
Navigation bar or buttons
Animations or transitions

Won’t Have

Backend/database functionality
Login or authentication system
Advanced JavaScript features
API integrations
Complex animations

5. Assumptions and Constraints
Assumes all team members will provide their own avatar images
Assumes beginner-level knowledge of HTML and CSS (no advanced JS)
Limited time, so focus is on basic functionality over advanced design
No backend—this is a static website only
File naming must follow required format (e.g., [name]_profile.html)
Collaboration depends on all members correctly pushing to GitHub
